Beware Python's pyc files

Published 2014-05-09 Modified 2014-05-09 Tags Python / pyc

A quick reminder to myself more than anything.

Had the issue of a strange ImportError: from itertools import tee. It's a little odd because itertools is a Python built-in module.

Running Python and doing the import from the directory where I was running code worked fine. However, running a script in that directory that was importing a package, that itself imported itertools gave this behaviour.

I wasn't really sure what was causing it, but on a whim I deleted the existing itertools.pyc file in that directory. When I looked, it turns out that itertools had a pyc file in the package directory. If you have inexplicable errors like this with no obvious explanation, it's worth perhaps deleting the cached .pyc files. That is, unless this applies to your coding workflow.
