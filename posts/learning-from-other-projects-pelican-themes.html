Learning from other projects: pelican-themes

Published 2019-12-13 Tags git / project management

Lack of updates

This blog, at the time of writing, uses Pelican to convert the Markdown blog content into HTML that I publish.

If you noticed from the dates on my posts, there has been a long time between the last post and this one. This was primarily because there was considerable work needed to get this blog setup up-to-date (upgrading Pelican to the latest version, and pulling in updates to the theme).

Updating the theme in particular was made slightly more complicated by the way the theme moved from its original repository and was "parented" by the pelican-themes repository, not as a submodule but as the de facto repository for that theme.
